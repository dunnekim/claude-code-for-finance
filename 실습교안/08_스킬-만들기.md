# Part 8: Skill 만들기

> **형태:** 실습 (25분)
>
> **목표:** Skill 개념 체험 → 직접 생성 → 테스트 → 수정

---

## Skill이란

**Skill = 나만의 슬래시 명령어**

Part 6~7에서 매번 입력했던 긴 프롬프트를 **한 줄로 압축**합니다.

```
Before:  "우리금융지주 종목코드 316140의 최근 3개년 연결 재무제표에서
          영업수익, 영업이익을 가져와서 재무비율 분석하고 저장해줘..."

After:   /재무분석 316140
```

| Skill의 가치 | 설명 |
|-------------|------|
| **반복 제거** | 긴 프롬프트를 매번 입력할 필요 없음 |
| **일관성** | 누가 실행해도 동일한 형식, 동일한 기준 |
| **공유 가능** | 파일 하나 공유하면 팀 전체가 사용 |
| **수정 가능** | 업무 요건이 바뀌면 텍스트 수정으로 즉시 반영 |

### 파일 위치 규칙

Skill 파일은 **두 곳** 중 하나에 저장할 수 있습니다:

| 위치 | 경로 | 적용 범위 |
|------|------|----------|
| **글로벌** (권장) | `~/.claude/skills/재무분석/SKILL.md` | 모든 프로젝트에서 사용 가능 |
| **프로젝트** | `프로젝트/.claude/skills/재무분석/SKILL.md` | 해당 프로젝트에서만 사용 |

> **Windows 경로:** `~/.claude/` = `C:\Users\사용자명\.claude\`

```
~/.claude/skills/재무분석/SKILL.md      ← 글로벌 (어디서든 /재무분석 사용)
프로젝트/.claude/skills/재무분석/SKILL.md  ← 프로젝트 전용
                ↑ 폴더명 = 명령어   ↑ 반드시 대문자
```

1. **폴더 이름 = 명령어 이름** — `재무분석/` → `/재무분석`
2. **파일 이름은 반드시 `SKILL.md`** (대문자)
3. **글로벌 위치(`~/.claude/skills/`)에 저장하면 어느 프로젝트에서든 사용 가능**
4. 프로젝트 `.claude/skills/`에 저장하면 해당 프로젝트 전용

> **주의:** 프로젝트 폴더 안의 `.claude/skills/`에만 넣으면, 다른 프로젝트에서는 `/` 목록에 나타나지 않습니다. **팀 공유나 범용 Skill은 글로벌 위치에 저장하세요.**

## 8-1: 예시 Skill 먼저 체험 (5분)

실습 폴더에 미리 배치된 예시 Skill을 먼저 실행해 봅니다. Skill이 어떻게 동작하는지 먼저 체감한 후, 직접 만드는 순서입니다.

### 미션 1: 예시 Skill 실행

채팅창에 `/`를 입력하면 명령어 목록이 뜹니다. `예시분석`이 보이면 선택하세요.

```
/예시분석 실습데이터/우리금융지주_연결재무제표.csv
```

Claude가 CSV 파일을 읽고, 사전 정의된 형식으로 분석 보고서를 생성합니다.

> **목록에 `/예시분석`이 안 보이면:**
> 1. `.claude/skills/예시분석/SKILL.md` 파일이 존재하는지 확인
> 2. 새 세션을 시작해 보세요 (채팅 패널에서 새 대화)
> 3. 그래도 안 되면 강의자에게 알려주세요

### 미션 2: 예시 Skill 파일 열어보기

```
.claude/skills/예시분석/SKILL.md 파일을 보여줘.
```

Skill 파일의 구조를 확인합니다:

| 부분 | 의미 |
|------|------|
| `---` ~ `---` | **Frontmatter** — Skill의 메타데이터 |
| `name` | Skill 이름 = 슬래시 명령어 |
| `description` | Claude가 이 Skill의 용도를 판단하는 기준 |
| `argument-hint` | 사용자에게 보이는 인자 힌트 |
| `$ARGUMENTS` | 명령어 뒤에 입력한 값이 들어가는 빈칸 |

`$ARGUMENTS`는 빈칸입니다. `/예시분석 파일.csv` → `$ARGUMENTS`에 `파일.csv`가 들어갑니다.

## 8-2: 직접 Skill 만들기 (10분)

이제 Part 6~7에서 반복했던 패턴을 Skill로 만듭니다.

### 미션 3: Claude에게 Skill 생성 요청

```
Part 6~7에서 5개사에 반복했던 작업 — OpenDart API로 기업 재무제표를 가져와서
재무비율 분석하고 보고서를 저장하는 것 — 을 Skill로 만들어줘.

요구사항:
- /재무분석 [종목코드] 형태로 실행
- 최근 3개년 연결 재무제표 자동 조회
- 금융업의 경우 영업수익 항목 사용 (CLAUDE.md 규칙 참고)
- 성장성/수익성/안정성/효율성 분석 자동 수행
- analysis/ 폴더에 마크다운 보고서 저장
- API 키는 환경변수 OPENDART_API_KEY 사용
- API가 안 되면 실습데이터/ 폴더의 CSV를 대체로 사용

~/.claude/skills/재무분석/SKILL.md에 저장해줘. (글로벌 위치)
```

## 8-3: Skill 테스트 (5분)

### 미션 4: 다른 기업으로 테스트

```
/재무분석 082640
```

(동양생명 — Part 7에서 한 번 분석했지만, 이제는 한 줄로)

**확인:**
- `/`를 입력했을 때 `재무분석`이 명령어 목록에 보이는가?
- Part 6~7에서 했던 것과 동일한 흐름으로 분석이 진행되는가?
- analysis/에 결과 파일이 저장되는가?
- 영업수익이 정확히 사용되었는가?

> **트러블슈팅:**
>
> | 증상 | 해결 |
> |------|------|
> | `/재무분석`이 목록에 안 보임 | 경로가 `.claude/skills/재무분석/SKILL.md`인지 확인 |
> | 파일명 문제 | `SKILL.md` 대문자인지 확인 |
> | frontmatter 오류 | `name:` 필드가 있는지 확인 |
> | 여전히 안 됨 | 새 세션 시작 후 재시도 |

## 8-4: Skill 수정 (5분)

### 미션 5: 분석 항목 추가

```
.claude/skills/재무분석/SKILL.md에서
분석 항목에 자기자본비율(자본총계 / 자산총계)을 추가해줘.
금융업에서 자본 적정성을 보는 핵심 지표라는 코멘트도 함께.
```

수정 후 다시 테스트:

```
/재무분석 316140
```

(우리금융지주 — 같은 회사를 다시 돌려서 새 항목이 보고서에 추가되었는지 확인)

> **핵심:** Skill은 고정된 프로그램이 아닙니다.
> 업무 요건이 바뀌면 SKILL.md의 **텍스트를 수정**하면 됩니다.
> 코드를 수정하는 것이 아니라 자연어 지침을 수정하는 것입니다.
>
> 이 파일을 팀원에게 공유하면, 누가 `/재무분석`을 실행해도 동일한 품질의 보고서가 나옵니다.
