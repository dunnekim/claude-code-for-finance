# Part 1: 사전 세팅 가이드

## 예상 소요 시간

약 30분 (OpenDart API Key 발급 포함)

## Step 1 — VS Code 설치

1. [code.visualstudio.com](https://code.visualstudio.com) 접속
2. **Windows** 버전 다운로드 → 설치 (기본 옵션 그대로)
3. 설치 후 VS Code를 한 번 실행하여 정상적으로 열리는지 확인

Microsoft에서 만든 무료 코드 에디터입니다. 워크숍 실습의 기본 환경입니다.

---

## Step 2 — Git + Python 설치 (터미널 한 번에)

Python 공식 사이트를 일일이 찾는 대신, Windows 터미널에서 **두 줄로** 설치합니다.

1. 시작 메뉴 → **"터미널"** 또는 **"PowerShell"** 검색 후 실행
2. 아래 명령어를 차례로 입력 (각 줄 끝에 Enter):

```
winget install Python.Python.3.12
winget install Git.Git
```

3. 설치 완료 후 **터미널을 완전히 닫고, 새 터미널 창을 다시 엽니다** (PATH 갱신을 위해 필수)
4. 아래 명령어로 정상 설치를 확인:

```
python --version
git --version
```

`Python 3.12.x`와 `git version 2.x.x`가 출력되면 성공입니다.

> **winget이 인식되지 않는 경우** (구형 Windows 또는 회사 IT 정책 차단 시):
>
> - **Python 수동 설치:** [python.org/downloads/release/python-3128](https://www.python.org/downloads/release/python-3128/) → Windows installer (64-bit) → 설치 화면에서 **"Add Python to PATH"** 체크 필수
> - **Git 수동 설치:** [git-scm.com/download/win](https://git-scm.com/download/win) → 64-bit Git for Windows Setup → 기본 옵션으로 설치
>
> 두 설치 후 터미널을 다시 열고 위 확인 명령어를 다시 실행하세요.

---

## Step 3 — Python 패키지 설치

DART API 호출과 웹 데이터 수집에 필요한 라이브러리를 설치합니다.

Step 2에서 사용한 같은 터미널 (또는 새 터미널)에서:

```
pip install requests beautifulsoup4
```

설치가 끝난 후 두 패키지가 정상 작동하는지 확인:

```
python -c "import requests; print('OK')"
python -c "import bs4; print('OK')"
```

각각 `OK`가 출력되면 성공입니다.

> **requests:** DART API 호출용 (HTTP 요청 라이브러리)
> **beautifulsoup4:** 웹 데이터 수집용 (HTML 파싱 라이브러리, Python 코드에서는 `bs4`로 import)

## ⭐ Step 5 — OpenDart API Key 발급 + 환경변수 등록 (가장 중요)

가장 중요한 단계입니다. API Key를 환경변수에 등록하지 않으면 워크숍 실습이 불가능합니다.

### 5-1. API Key 발급

1. [opendart.fss.or.kr](https://opendart.fss.or.kr) 접속 → 회원 가입
2. 로그인 후 **[인증키 신청/관리]** → **[인증키 신청]** 메뉴
3. 이름·연락처·사용 목적("학습용 재무 데이터 조회") 입력 후 신청
4. **즉시 발급됩니다.** 발급된 인증키를 메모장에 복사해 두세요.

### 5-2. 환경변수 등록

Windows 터미널에서 아래 명령어를 입력 (`발급받은_키` 부분에 실제 키를 붙여넣기):

```
setx OPENDART_API_KEY "발급받은_키"
```

`SUCCESS: Specified value was saved.` 가 출력되면 성공입니다.

### 5-3. 등록 확인

**터미널을 완전히 닫고 새 터미널을 열어서** 아래 명령어로 확인 (환경변수는 새 프로세스부터 적용됨):

```
echo %OPENDART_API_KEY%
```

발급받은 키가 화면에 출력되면 성공입니다.

> **왜 환경변수에 저장하나요?**
> API Key를 코드에 직접 넣으면 실수로 GitHub 등에 공유될 수 있습니다.
> 환경변수를 쓰면 코드에는 변수 이름만 남고, 실제 키는 본인 PC에만 존재합니다. 보안 습관의 시작입니다.
