# Part 5: Claude Code 핵심 기능

> **형태:** 실습 (30분)
>
> **목표:** 파일 참조, 환경변수 활용, 권한 모드, CLAUDE.md 작성

---

## 5-1: 기본 대화와 파일 읽기 (10분)

### 미션 1: 자기소개

```
나는 회계법인에서 일하는 회계사야.
주로 IFRS 기준 재무제표 분석, 밸류에이션, FDD를 수행해.
한국어로 대화하자.
```

### 미션 2: 파일 읽기

```
실습데이터 폴더에 있는 CSV 파일들을 읽고, 어떤 데이터가 들어있는지 요약해줘.
```

### 미션 3: @ 참조

`@`를 입력하면 파일 경로를 자동완성할 수 있습니다.

```
@실습데이터/우리금융지주_연결재무제표.csv 이 파일에서 영업수익 추이를 분석해줘.
```

> **TIP:** `@` 입력 후 파일명 일부를 치면 자동완성 후보가 뜹니다. `Tab`으로 선택.
>
> **참고:** 금융업은 일반 제조업과 달리 "매출액"이 아닌 "영업수익"이라는 항목을 사용합니다. 이자수익, 수수료수익, 보험료수입, 자기매매익 등이 영업수익에 포함됩니다.

## 5-2: 권한 모드 이해 (5분)

`Shift+Tab`을 누르면 권한 모드가 순환합니다.

| 모드 | 설명 | 추천 상황 |
|------|------|----------|
| **Ask before edits** | 매 작업마다 승인 | 처음 사용, 민감 작업 |
| **Edit Automatically** | 파일 수정 자동 승인 | 신뢰할 수 있는 반복 작업 |
| **Plan mode** | 분석만 가능, 수정 불가 | 코드 리뷰, 구조 파악 |

### 미션 4: 모드 전환

`Shift+Tab`으로 모드를 순환해 보고, **Ask before edits**로 돌아옵니다.

---

## 5-3: 환경변수 활용 (5분)

### 미션 5: API Key 확인

```
내 시스템 환경변수에 OPENDART_API_KEY가 등록되어 있는지 확인해줘.
키 값은 보여주지 말고, 존재 여부만 알려줘.
```

> **"키 값은 보여주지 말고"** — API Key가 대화 로그에 남는 것을 방지합니다. 보안 습관입니다. 시스템 환경변수는 운영체제가 프로그램에 전달하는 설정값으로, OS 레벨에서 키=값 형태로 저장되는 전역 설정이며 모든 프로그램이 참조할 수 있는 공통 정보입니다.

### 미션 6: API 연결 테스트

```
OPENDART_API_KEY 환경변수를 사용해서
OpenDart API 연결이 되는지 간단히 테스트해줘.
우리금융지주(316140)의 기업개황 정보를 가져와봐.
```

Claude가 Python 스크립트를 작성·실행하여 API 응답을 받아오면 성공입니다.

## 5-4: CLAUDE.md에 업무 규칙 쌓기 (10분)

Part 3에서 확인한 CLAUDE.md에는 보안 규칙만 들어있었습니다. 이제 여기에 **업무 규칙을 추가**합니다. 보안 규칙 위에 업무 규칙을 쌓아가는 것 — 이것이 CLAUDE.md가 한 작업의 일관성을 만드는 방식입니다.

### 미션 7: 프로젝트 규칙 추가

```
CLAUDE.md 파일에 아래 내용을 추가해줘. 기존 보안 규칙은 그대로 유지하고 아래에 추가:

# 프로젝트 규칙
- 한국어로 응답
- 금액 단위: 백만원 (소수점 이하 반올림)
- 금융업(은행·보험·증권·금융지주)의 매출은 "영업수익" 항목으로 표시. 일반 제조업의 "매출액"과 구분.
- 재무비율 계산 시 IFRS 기준 적용 (ROE = 당기순이익 / 자본총계, ROA = 당기순이익 / 자산총계)
- OpenDart API 키는 환경변수 OPENDART_API_KEY 사용 (코드에 직접 입력 금지)
- OpenDart API로 기업 고유번호(corp_code)가 필요할 때, 실습데이터/CORPCODE.xml 파일을 먼저 참조할 것 (API 재다운로드 불필요)
- 분석 결과는 analysis/ 폴더에 마크다운으로 저장
- 파일명 규칙: [기업명]_[분석유형].md (예: 우리금융지주_재무분석.md)
```

> **왜 영업수익 규칙이 핵심인가?** 금융업과 제조업은 매출의 의미와 구성이 완전히 다릅니다. 우리금융지주의 "영업수익"은 이자수익 + 수수료수익 + 기타이고, 카카오뱅크도 마찬가지입니다. 만약 이 규칙이 없으면 AI가 "매출액" 항목을 찾아 0원으로 보고하거나, 잘못된 항목을 가져올 수 있습니다. **이 한 줄이 5개사 분석의 정합성을 결정합니다.**

### 미션 8: 적용 확인

새 대화를 시작하고:

```
우리금융지주 2023년 영업수익이 30조원이야. 우리 프로젝트 단위로 변환해줘.
```

Claude가 **백만원 단위**로 변환하면 CLAUDE.md가 적용된 것입니다.

> **핵심:** 이 규칙 파일을 팀원에게 공유하면, 누가 Claude Code를 쓰든 동일한 기준이 적용됩니다. 한 작업의 일관성을 만드는 첫 번째 조각입니다.
