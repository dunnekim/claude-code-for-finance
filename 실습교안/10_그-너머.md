이것이 한 작업의 자동화 패턴입니다. **첫 번째 블록**입니다.

---

## 더 쌓을 수 있는 명령어들

오늘 만든 Skill 패턴(CLAUDE.md + Skill + API)을 그대로 적용하면 다음과 같은 작업도 한 줄 명령어로 만들 수 있습니다.

### 1. 비교 분석

```
/비교분석 316140 105560 055550 086790
```

복수 종목코드 → 각각 데이터 수집 → 비교 테이블 → 종합 의견까지 자동 생성. Part 7에서 수동으로 5번 반복했던 작업의 압축판입니다.

### 2. 컨센서스 조회

```
/컨센서스 316140
```

네이버금융에서 증권사 컨센서스(목표주가, 투자의견, 실적 추정치)를 가져와 정리합니다. DART 재무 실적과 증권사 추정치를 나란히 놓으면, 시장 기대 대비 실적 괴리를 한눈에 볼 수 있습니다.

> 사전 제작 스크립트가 `scripts/fetch_naver_consensus.py`에 포함되어 있습니다. 워크숍 후 직접 시도해 보세요.

### 3. PBR 자동 계산

```
/PBR분석 316140
```

DART 재무(자본총계) + 네이버금융 시가총액을 결합하여 PBR을 자동 계산합니다. 두 개의 다른 데이터 소스를 결합하는 패턴을 보여주는 좋은 예시입니다.

```
PBR = 시가총액 / 자본총계
```

이 한 줄의 공식을 위해 실제로는: ① DART에서 corp_code 조회, ② DART에서 자본총계 추출, ③ 네이버금융에서 시가총액 스크래핑, ④ 계산, ⑤ 저장. 다섯 단계가 한 줄 명령어 뒤에 숨어 있습니다.

### 4. 공시 조회

```
/공시조회 316140 2025-01-01 2025-12-31
```

특정 기간의 공시 자동 수집 + 키워드 필터링.

### 5. peer 확장

```
/peer확장 316140
```

오늘 만든 `/피어스크리닝`의 강화판. 사용자가 후보를 입력하지 않아도, 시가총액·업종 코드로 1차 필터링한 후 상위 30~50개를 자동으로 비교합니다.

## 구조 자체의 확장

위 5개는 모두 "Skill 하나 추가"입니다. 그보다 더 큰 확장은 **구조 자체의 변경**입니다.

### 1. Sub-agent — 역할 분리

Claude Code는 별도의 AI 에이전트를 생성하여 작업을 위임할 수 있습니다. 실습 폴더의 `.claude/agents/`에 예시가 배치되어 있습니다.

```
.claude/agents/
├── 데이터수집가.md      ← API 호출 + 데이터 정리
└── 재무분석가.md        ← 재무비율 계산 + 해석
```

이렇게 하면 5개사 데이터를 **병렬로 동시에** 수집할 수 있습니다. 순차적으로 10분 걸릴 작업이 3분에 끝납니다.

### 2. MCP — 외부 서비스 연동

| 연동 대상 | 가능한 작업 |
|----------|------------|
| Google Sheets | 분석 결과를 스프레드시트에 자동 저장 |
| Notion | 보고서를 데이터베이스에 아카이빙 |

### 3. Hooks — 자동 트리거

특정 이벤트에 자동으로 반응합니다.

- 세션 시작 시 → 오늘의 공시 목록 자동 조회
- 분석 완료 시 → 결과 자동 요약
- 세션 종료 시 → 작업 내역 자동 기록

---

## ⚠ 회사 PC 활용 제한

위 세 가지 심화 기능은 회사 환경에서 다음과 같은 제약이 있을 수 있습니다.

- **Sub-agent**: 본 환경에서 작동합니다. 큰 제약 없음.
- **MCP**: 외부 서비스(Google Sheets, Notion 등)에 인증·연결이 필요합니다. 회사 IT 정책으로 차단될 수 있습니다.
- **Hooks**: 백그라운드 자동 실행을 포함합니다. 보안 정책상 제한될 수 있습니다.

본격 도입 전에 IT 보안 부서와 상의하시고, 개인 환경(집/노트북)에서 먼저 실험해 보기를 권장합니다.

|------|------|------|

## 마무리

오늘 3시간 20분 동안 한 일:

1. **왜 AX인가** — 한 작업의 자동화에서 시작한다
2. **보안 설계** — 리서치 전용 환경
3. **핵심 기능** — CLAUDE.md, @ 참조, 환경변수, 권한 모드
4. **DART 1개사 분석** — OpenDart API + 재무비율
5. **DART 5개사 비교** — 같은 패턴 5번 반복 인지
6. **Skill 만들기** — `/재무분석` 한 줄 자동화
7. **패턴 응용** — `/피어스크리닝`으로 같은 패턴을 새 도메인에 적용

오늘은 한 작업을 자동화했습니다. 같은 패턴으로 비교분석, 컨센서스, peer 스크리닝을 만들면, 결국 **팀의 반복 업무 전체가 한 줄 명령어**가 됩니다. 오늘 만든 `/재무분석`과 `/피어스크리닝`은 그 환경의 첫 번째 블록입니다.
