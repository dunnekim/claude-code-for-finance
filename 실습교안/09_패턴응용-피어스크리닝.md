# Part 9: 패턴 응용 — 피어 스크리닝

> **형태:** 실습 (15분)
>
> **목표:** Part 8에서 만든 Skill 패턴을 새 도메인(피어 스크리닝)에 적용

---

## 왜 이 파트가 워크숍의 클라이맥스인가

Part 8에서 만든 `/재무분석`은 **한 작업**의 자동화입니다. 그러나 진짜 가치는 그 작업 하나가 아닙니다. **같은 패턴**(CLAUDE.md + Skill + API)을 다른 반복 업무에 적용할 수 있다는 것입니다.

이번 파트에서 직접 증명합니다.

새 작업: **"타겟 회사와 사업 모델이 비슷한 회사를 찾는다"** (Peer Screening)

이건 비교분석, 컨센서스 수집과 함께 PE/FDD에서 자주 반복되는 작업입니다. 보통 시가총액·업종 코드로 1차 필터링한 후 사업의 내용을 일일이 읽어서 사람이 정성 판단하는데, 시간이 많이 듭니다.

오늘 만든 Skill 패턴을 그대로 적용해서, **이 작업도 한 줄 명령어로** 만듭니다.

---

## 9-1: 개념 설명 (3분)

### 피어 스크리닝의 본질

```
타겟 회사 → 후보 회사들 → 사업의 내용 비교 → 유사도 점수 → 순위
                                              (AI가 텍스트 비교)
```

DART의 사업보고서에는 "사업의 내용"이라는 섹션이 있습니다. 회사가 어떤 사업을 하는지 자세히 설명한 텍스트입니다. 이 텍스트를 비교하면 사업 모델 유사성을 정량화할 수 있습니다.

### 왜 후보를 직접 입력하는가

이상적으로는 KOSPI 3500개 상장사 중에서 자동으로 후보를 찾는 게 좋습니다. 하지만 그러면 3500번 API 호출이 필요해 워크숍 시간 안에 불가능합니다.

대신 **사용자가 후보 5~10개를 직접 입력**하는 방식으로 단순화합니다. 후보 리스트는 사람이 1차 필터(시가총액, 업종)로 만들고, AI는 그 안에서 사업 내용을 비교하는 역할만 합니다.

이게 현실적인 FDD/Valuation 워크플로우입니다.

## 9-2: /피어스크리닝 Skill 만들기 (10분)

### 미션 1: Skill 생성 요청

```
새 Skill을 만들어줘.

요구사항:
- 이름: /피어스크리닝
- 사용법: /피어스크리닝 [타겟종목코드] [후보종목코드1] [후보종목코드2] ...
- 동작:
  1. 타겟 회사와 모든 후보 회사의 사업의 내용을 OpenDart API로 가져온다
     (사업보고서 첨부문서에서 추출, document API 사용)
  2. AI가 사업 내용을 텍스트 유사도로 비교한다
  3. 유사도 순으로 정렬해서 표로 출력
  4. 각 회사별로 사업 모델의 핵심 차이점을 한 줄씩 코멘트
  5. analysis/[타겟종목코드]_피어스크리닝.md에 저장
- API 키는 환경변수 OPENDART_API_KEY 사용
- 사업의 내용 텍스트가 너무 길면 핵심 부분(주요 사업, 주요 제품/서비스)만 발췌

.claude/skills/피어스크리닝/SKILL.md에 저장해줘.
```

> **여기서 Part 8의 패턴이 그대로 적용되는 것을 확인하세요.**
> Skill 폴더 구조, frontmatter, $ARGUMENTS 사용법, CLAUDE.md 규칙 — 모든 것이 Part 8과 동일합니다. 단지 분석 대상만 다릅니다.

## 9-3: 실제 우리금융지주 피어 스크리닝 (5분)

### 미션 3: 결과 검증

생성된 `analysis/316140_피어스크리닝.md`를 열어 결과를 확인합니다.

**예상 결과 (대략):**
- 1~3위: KB금융, 신한지주, 하나금융지주 (같은 SIFI 은행지주, 가장 유사)
- 4~5위: BNK·JB금융지주 (지방 은행지주, 사업 모델은 비슷하지만 규모·지역 차이)
- 6~9위: 메리츠금융지주, 한국금융지주, iM금융지주, 등 (증권지주는 사업 모델이 더 다름)

**결과가 이렇게 카테고리별로 자연스럽게 분류되면 성공입니다.** AI가 사업의 내용 텍스트를 제대로 비교하고 있다는 증거입니다.

## 패턴이 정말 재사용되는가 — 핵심 질문

방금 만든 `/피어스크리닝` Skill을 보세요.

- **CLAUDE.md 규칙은?** 그대로 (한국어, 단위, 영업수익, 환경변수, 저장 위치)
- **Skill 파일 구조는?** Part 8과 동일 (frontmatter, $ARGUMENTS, 절차 텍스트)
- **API 사용법은?** 동일 (환경변수, corp_code 조회 → 실제 데이터 조회)
- **저장 위치는?** 동일 (analysis/ 폴더, 마크다운)

**달라진 것:** 분석 대상이 재무제표가 아니라 사업의 내용 텍스트라는 점뿐입니다.

이것이 워크숍의 핵심 메시지입니다. **한 패턴을 익히면, 다른 작업도 30분 안에 만들 수 있습니다.** 이 시간을 한 번 투자하면 그 다음 작업, 또 그 다음 작업이 모두 한 줄 명령어가 됩니다.
