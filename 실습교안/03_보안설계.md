# Part 3: 보안 설계 — 리서치 전용 환경

> **형태:** 강의 + 실습 (10분)
>
> **목표:** "클라이언트 자료 유출 걱정" 해소, 안전한 작업 환경 구성

---

## 왜 이 파트가 가장 먼저인가

회계법인에서 일하는 사람이 AI 도구를 처음 접할 때 가장 먼저 드는 생각:

> "이거 클라이언트 자료가 유출되는 거 아니야?"

이 질문에 명확한 답이 없으면 나머지 3시간 동안 아무것도 시도하지 않습니다.

**결론부터:** 올바르게 설정하면 안전합니다. 핵심은 **"무엇을 넣느냐"**입니다.

---

## 원칙: 폴더 격리

Claude Code는 **VS Code에서 열린 폴더**를 프로젝트 루트로 인식합니다.

**어떤 폴더를 여느냐**가 보안의 첫 번째 방어선입니다.

### 절대 규칙

| ✓ 넣어도 되는 것 | ✗ 절대 넣지 않는 것 |
|------------------|---------------------|
| DART 공시 데이터 (공개) | 회사자료 |
| IFRS 기준서 (공개) | 클라이언트 재무모델 |
| 금감원 전자공시 (공개) | 실사 자료 |
| 본인이 작성한 분석 템플릿 | 계약서 / SPA |
| 연습용 샘플 데이터 | 고객 PII |

---

## 4중 방어 체계

### 1. 물리적 격리 — 전용 폴더

`실습환경/` 폴더만 사용합니다. 고객자료, 클라이언트 폴더는 절대 열지 않습니다.

### 2. 권한 승인 — 매번 확인

Claude Code는 파일을 읽거나 명령어를 실행할 때 **반드시 승인을 요청**합니다.

| 선택 | 의미 |
|------|------|
| **Yes** | 이번 한 번만 승인 |
| **Yes, always allow** | 이 프로젝트에서 해당 작업 항상 승인 |
| **No** | 거부 |

모르는 경로나 예상 밖의 명령이 뜨면 **No**를 누르세요.

### 3. CLAUDE.md — AI의 행동 규범

프로젝트 루트의 `CLAUDE.md`에 보안 규칙을 명시하면 Claude가 세션 시작 시 자동으로 읽고 준수합니다.

### 4. 입력 통제 — 클라이언트 자료를 프롬프트에 넣지 않음

API 호출과 파일 작업은 로컬에서 일어납니다. Anthropic 서버로 전송되는 것은 **대화 내용(프롬프트와 응답)** 입니다. 클라이언트 자료를 프롬프트에 붙여넣지 않는 것이 가장 중요한 원칙입니다.

---

## 실습: CLAUDE.md 보안 규칙 확인

실습 폴더에 미리 포함된 `CLAUDE.md`를 열어 내용을 확인합니다.

```markdown
# 보안 규칙 (절대 준수)
- 이 프로젝트 폴더 외부의 파일을 절대 읽거나 접근하지 마세요
- 절대경로(C:\Users 등)로 다른 디렉토리에 접근하지 마세요
- cd .. 등으로 상위 폴더로 이동하지 마세요
- 이 환경은 공개 데이터 리서치 전용입니다
```

지금은 보안 규칙만 적혀 있습니다. Part 5에서 여기에 **업무 규칙을 추가**하면서, 이 파일이 팀의 공통 기준이 되어가는 과정을 체험합니다.

> **이것은 기술적 차단이 아닌 AI에 대한 지시입니다.**
> Claude는 이 규칙을 높은 확률로 준수합니다.
> 설사 외부 접근을 시도하더라도, 권한 승인 단계에서 직접 거부할 수 있습니다.
